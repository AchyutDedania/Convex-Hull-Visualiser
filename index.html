Jarvis March Algorithm

Jarvis March, also known as the Gift Wrapping Algorithm, is a simple and intuitive method for finding the convex hull of a set of points in two-dimensional space. It was proposed by R. A. Jarvis in 1973. The algorithm iteratively constructs the convex hull by finding the next point on the hull based on the current point and the remaining unprocessed points. The main idea behind the algorithm is to start from the point with the lowest y-coordinate (or the leftmost point in case of a tie) and then iteratively select the point that forms the smallest counterclockwise angle with the current point. This process continues until the algorithm reaches the starting point again, completing the convex hull. Jarvis March is relatively easy to implement and understand, making it a popular choice for educational purposes and small-scale applications. However, its time complexity is O(nh), where n is the number of input points and h is the number of points on the convex hull. In worst-case scenarios, where all points lie on the convex hull, Jarvis March becomes less efficient compared to more sophisticated algorithms like Graham Scan or Quickhull.

Kirkpatrick-Seidel Algorithm

The Kirkpatrick-Seidel algorithm is a widely used method for computing the convex hull of a set of points in two-dimensional space. It was introduced by James R. Kirkpatrick and Raimund Seidel in 1986. The algorithm builds upon the concept of divide-and-conquer, efficiently reducing the problem's complexity to achieve a time complexity of O(n log h), where n is the number of input points and h is the number of points on the convex hull. At its core, the algorithm involves recursively partitioning the set of points into smaller subsets, computing the convex hull of each subset, and then merging these convex hulls to form the final convex hull. This merging step is where much of the algorithm's efficiency lies, as it employs techniques to eliminate unnecessary points from consideration. The Kirkpatrick-Seidel algorithm has been proven to be one of the most efficient methods for computing convex hulls in two dimensions, offering significant improvements over earlier algorithms in terms of both time complexity and practical performance.

Analysis of Different Algorithms

Kirkpatrick-Seidel

Time Complexity Analysis:

The time complexity analysis of the Kirkpatrick-Seidel algorithm revolves around its divide-and-conquer approach and the efficiency of merging convex hulls. Here's a breakdown of how the O(n log h) time complexity is achieved:

  • Divide-and-Conquer: The algorithm recursively divides the set of points into smaller subsets. This step has a time complexity of O(log n) because each recursion level divides the problem size by a constant factor.
  • Convex Hull Computation: At each recursion level, the convex hull of the subset of points needs to be computed. The time complexity of computing the convex hull of a subset of points is O(m log m), where m is the number of points in the subset. However, because the number of points in each subset is not necessarily fixed and can vary, we denote the number of points in the convex hull of a subset as h, which is typically smaller than m. Thus, the time complexity of computing the convex hull of each subset becomes O(n log h) in total for all subsets.
  • Merging Convex Hulls: After computing convex hulls for individual subsets, they need to be merged. The merging process involves merging two convex polygons, which can be done in linear time, i.e., O(h). Since there are O(log n) levels of recursion and at each level, merging takes O(h) time, the total time complexity for merging is O(h log n).

Putting it all together, the time complexity of the Kirkpatrick-Seidel algorithm can be expressed as:

O(Divide-and-Conquer + Convex Hull Computation + Merging Convex Hulls) = O(log n + n log h + h log n)

Since h is generally smaller than n (h ≤ n), the time complexity simplifies to: O(n log h)

This analysis shows that the Kirkpatrick-Seidel algorithm achieves a time complexity of O(n log h), which is optimal for computing the convex hull of a set of n points in two-dimensional space.

Jarvis March

Time Complexity Analysis:

The time complexity analysis of the Jarvis March algorithm involves understanding its iterative process and how it selects points to construct the convex hull. Here's how the O(nh) time complexity is derived:

  • Iteration: The algorithm iterates through each point in the input set to find the next point on the convex hull. In the worst case, each point may need to be considered for inclusion in the convex hull. This results in a time complexity of O(n), where n is the number of input points.
  • Point Selection: For each point considered in the iteration, the algorithm must find the next point that forms the smallest counterclockwise angle with the current point. This involves comparing each remaining point against the current point to determine the angle. In the worst case, this comparison operation takes O(h) time, where h is the number of points on the convex hull found so far.
  • Overall Complexity: Since the algorithm iterates through each point and, for each point, potentially compares it with each other remaining point to find the next hull point, the overall time complexity is the product of the number of points (n) and the maximum number of comparisons needed at each step (h). Thus, the time complexity is O(nh).

In practice, the actual time complexity can vary depending on the distribution of points and their arrangement, but in the worst case, Jarvis March exhibits a time complexity of O(nh). This makes it less efficient compared to algorithms like Graham Scan or Quickhull, especially when the number of points on the convex hull is proportional to the number of input points.
